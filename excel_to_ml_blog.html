How I Went from Excel Sheets to Machine Learning Models

Back in the day I thought Excel was everything. I had multiple spreadsheets, pivot tables, VLOOKUPs, macros and all that. I was comfortable. But when I heard about machine learning models doing predictions and insights on data, I felt a mix of excitement and “can I do this?”-fear. The shift from Excel to ML was not instant, and it’s not perfect even now, but the journey taught me a lot.

The first step was accepting the limits of Excel. Spreadsheets are fine when you have thousands of rows, but when you hit tens of thousands, or when you want to automate tasks, connect to databases or build a prediction model, it gets messy. I realized I needed a tool that could scale, that could run the same steps reliably, and handle data pipelines. That’s when I picked up Python and its libraries like pandas, numpy, scikit-learn. I started by exporting my Excel data into .csv files, reading them in pandas, doing the same cleaning I did in Excel but in code: dropping nulls, filling values, converting dates, merging tables.

Once I was comfortable cleaning and transforming data in Python, I moved to building a basic machine learning model. My first model was simple: I took my cleaned data, split into train and test sets, used a logistic regression or decision tree (whichever made sense), trained it, predicted the outcome, evaluated accuracy. It felt magical the first time I saw the model giving a prediction rather than just a number in a cell. I built code that took what I used to do manually in Excel like calculate metrics, filter groups, chart results and automated it so every time new data arrived I could run the whole thing and get results in minutes instead of hours.

During this transition I learned that the mindset changes. In Excel I thought row by row, sheet by sheet. In machine learning I think in pipelines: source data → clean → transform → feature engineering → model → evaluate → deploy. I learned concepts like overfitting, regularization, train/test split, cross-validation. I saw that it’s not just about getting a result, but getting a result that generalizes to new data, and saying “yes this model works” not just “it worked once for this file”.

Today, when I build a project I almost always start with “what did I used to do in Excel?” and ask “how can I code this, make it automatic, scale it, wrap it in a pipeline or notebook, then experiment models”. And yes, I still open Excel for small tasks, but I no longer live there. I moved to code-first, to workflows, to machine learning. So if you are stuck in spreadsheets and wondering if you can make the jump, the answer is yes. You already have the foundations: cleaning data, logic, understanding metrics. Just take one code step, one project step, build a model, and you’ll see the difference.
